# Contributing to NiftyNest

Thank you for your interest in NiftyNest! We welcome contributions that help improve this architecture template for the Flutter community.

## How to Contribute

1. **Fork the repository** and create your branch from `main`.
2. **If you've added code** that should be tested, add tests.
3. **Ensure the test suite passes** and `flutter analyze` returns zero issues.
4. **Follow the existing architecture** (Feature-First, Riverpod Notifiers).
5. **Issue a pull request!**

## Coding Standards

- Use **Feature-First Architecture** for all new modules.
- Maintain **strict separation** between presentation and application layers.
- Use **AppDesign tokens** for all styling (spacing, radius, typography).
- Ensure all new interactive elements have proper **Semantics** for accessibility.

## Commit Message Guidelines

We follow a professional commit strategy:
- `feat`: A new feature
- `fix`: A bug fix
- `docs`: Documentation only changes
- `style`: Changes that do not affect the meaning of the code (white-space, formatting, etc)
- `refactor`: A code change that neither fixes a bug nor adds a feature
- `perf`: A code change that improves performance
